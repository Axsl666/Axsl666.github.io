操作系统 物理内存管理：非连续内存分配

非连续内存分配

非连续内存分配需求背景

  • 连续分配的缺点
    • 分配给程序的物理内存必须连续
    • 存在内、外碎片
    • 内存分配的动态修改困难
    • 内存利用率低
  • 非连续分配的设计目标：提高内存利用效率和管理灵活性
    • 允许一个程序使用非连续的物理地址空间
    • 允许共享代码与数据
    • 支持动态加载和动态链接

实现

  • 非连续分配需要解决的问题
    • 虚拟地址和物理地址的转换
      • 软件实现（灵活、开销大）
      • 硬件实现（够用，开销小）
  • 非连续分配的硬件辅助机制
    • 如何选择非连续分配中的内存分块大小
      • 段式存储管理 segment
      • 页式存储管理 page

段地址空间

  • 进程的段地址空间由多个段组成：主代码段、子模式代码段、共用库代码段、堆栈段stack、数据段heap、初始化数据段、符号表等等
  • 段式存储管理的目的：更细的粒度和灵活的分离与共享

段访问机制

机制

  • 段的概念
    • 段表示访问方式和存储数据等属性相同的一段地址空间
    • 对应一个连续的内存块
    • 若干个段组成进程的逻辑地址空间
  • 段访问：逻辑地址由二元组（s，addr）表示
    • s——段号
    • addr——段内偏移

页式存储管理

页式存储管理概念

  • 页帧（帧，物理页面，Frame，Page Frame）
    • 把物理地址划分为大小相同的基本分配单位
    • 大小为2的n次方，一般为4kb。
  • 页面（页、逻辑页面，Page）
    • 把逻辑地址空间划分为大小相同的基本分配单位
    • 帧和页的大小必须是相同的
  • 页面到页帧的转换
    • 逻辑地址到物理地址
    • 页表
    • MMU/TLB

帧 Frame

  • 物理内存被划分为大小相同的帧
  • 内存的物理地址表示：二元组（f，o）
    • f——帧号（F位，共有 2^F^ 个帧）
    • o——帧内偏移 (S位，每帧由2^S^字节)
  • 物理地址 = f*2^s^ +o

页 Page

  • j进程逻辑地址空间被划分为大小相同的页
    • 页内偏移 = 帧内偏移
    • 通常页号 != 帧号
  • 内存的物理地址表示：二元组（p，o）
    • p——页号（P 位，共有 2^P^ 个页）
    • o——页内偏移 (S位，每帧由2^S^字节)
  • 物理地址 = p*2^s^ +o

页式存储管理地址转换

地址映射

  • 物理地址中的帧号不连续
  • 逻辑地址中的页号连续
  • 不是所有的页都有对应的帧

页表

页表概述

页表概述

页表结构

  • 每一个进程都有一个页表

    • 每个页面对应与一个页表项
    • 随着进程运行状态而动态变化
    • 页表基址寄存器（PTBR：Page Table Base Register）

  • 页表项组成

    • 帧号：f
    • 页表标志项
      • 存在位 resident bit
      • 修改位 dirty bit
      • 引用位 clock/reference bit
      • 等等

页式存储管理的性能问题

  • 内存访问性能问题
    • 访问一个内存单元需要两次内存访问
      • 第一次：获取页表项
      • 第二次：访问数据
  • 页表大小问题
    • 页表可能非常大
    • 64位机器
  • 处理方法
    • 缓存（Caching）：减少内存访问
    • 间接（Indirection）访问：减少页表长度

快表 Translation Look-aside Buffer，TLB

  • 缓存近期访问的页表项
    • TLB使用关联存储（associative memory）实现，具备快速访问性能
    • 如果TLB命中，物理页号可以很快被获取
    • 如果TLB未命中，对应的表项被更新到TLB中

多级页表

  • 通过间接引用将页号分成k级
    • 建立页表树
    • 减少每级页表的长度
    • 访问k+1次内存

一般来说不会用到全部的逻辑地址空间，所以通过上级页表存在位，省去不存在的页表，节省了物理内存。

反置页表

  • 对于大地址空间（64-bits）系统，多级页表变得繁琐
    • 比如5级页表
    • 逻辑（虚拟）地址空间的增长速度快于物理地址空间
  • 页寄存器和反置页面
    • 不让页表与逻辑地址空间的大小相对应
    • 让页表与物理地址空间的大小相对应

页寄存器

  • 每个帧与一个页寄存器（Page Register）关联，寄存器内容包括
    • 使用位（Residence bit）：此帧是否被进程占用
    • 占用页号（Occupier）：对应的页号
    • 保护位（Protection bits）
  • 页寄存器示例
    • 物理内存大小 4096 * 4096 = 4K * 4KB = 16MB
    • 页面大小：4KB
    • 页帧数：4096 = 4K
    • 页寄存器使用空间（假设每个页寄存器占8字节）
      • 8B *4K = 32KB
    • 页寄存器带来的额外开销： 32K / 16M = 0.2%（大约）
    • 虚拟内存大小：任意
页寄存器方案的特征
  • 优点
    • 页表大小相对于物理内存而言很小
    • 页表大小与逻辑地址空间大小无关
  • 缺点
    • 页表信息对调后，需要依据帧号找到页号，所以必须在页寄存器中搜索逻辑地址中的页号，直接遍历搜搜比较困难
页寄存器中的地址转换
  • CPU生成的逻辑地址找到对应物理地址的方法：
    • 对逻辑地址进行Hash映射，以减小搜索范围
    • 需要解决可能的冲突
  • 用快表缓存页表项后的页寄存器搜索步骤
    • 对逻辑地址进行Hash变换
    • 在快表中查找对应页表项
    • 有冲突时遍历冲突链表
  • 快表的限制
    • 快表的容量限制
    • 快表的功耗限制（StrongARM上快表功耗占27%）

反置页表 Invert Page Table

  • 基于Hash映射值查找对应页表项中的帧号
    • 进程标识与页号的Hash值可能有冲突
    • 页表项中包括保护位、修改位、访问位和存在位等标识
